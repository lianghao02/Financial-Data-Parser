# 專案特定細則：金融資料 CSV 轉 Excel 工具 (v1.4)

> [!IMPORTANT]
> 本專案受「全域開發憲法 v1.4」規範。所有開發計畫與 UI 介面必須 100% 使用台灣繁體中文。

## 1. 核心業務邏輯 (Financial Data Logic)
- **帳號強型別保護**：
  - 強制將所有帳號、身分證字號、銀行代碼識別為「字串 (String)」。
  - 使用 SheetJS 的 `t: 's'` 屬性寫入單元格，嚴禁出現科學記號或移除前導零。
- **智慧金額清理 (Regex Logic)**：
  - 必須過濾 `$`, `+`, `,` 及等號。
  - 特別針對 `.0+` 後綴（如 `.00`, `.000`）進行全數清空，轉換為純整數。
- **ZIP 遞迴處理**：
  - 支援前端解壓縮（使用 JSZip CDN）。
  - 若偵測到 ZIP 內還有 ZIP，必須進行遞迴提取，直到取得所有目標 CSV/Excel 檔案。

## 2. 匯出模式實作規範
- **個別儲存**：一對一產生檔案。
- **分頁合併**：多對一，每個檔案一個 Sheet。分頁命名遵循 `自訂前綴 + 序號`。
- **單頁堆疊 (Vertical Stack)**：
  - 多對一，所有資料集中在同一個 Sheet。
  - **自動除重**：僅保留第一份檔案的標題列，後續檔案僅提取資料列。

## 3. UI 配置與持久化 (Constitution Alignment)
依據憲法第 3 條，必須在網頁右側實作設定面板，包含以下連動功能：
- **參數設定**：
  - `EXCEL_FORMAT`: 下拉選單 (文字/數值/自動)。
  - `CLEAN_AMOUNT`: 開關 (移除符號與小數點)。
  - `MERGE_MODE`: 下拉選單 (個別/分頁/單頁堆疊)。
- **持久化要求**：
  - 所有 UI 調整後的數值必須存入 `localStorage`，重新整理頁面後需自動回填。
- **主題切換**：提供 `THEME_COLOR` 選擇器，並將色碼應用於 CSS 變數 `--primary-color`。

## 4. 效能與防禦性設計
- **載入遮罩**：處理 50MB 以上檔案或解壓縮時，必須顯示全螢幕 Loading 遮罩。
- **錯誤提示**：CSV 解析失敗時，必須彈出包含錯誤行數的繁體中文提示（使用 ⚠️ 標籤）。
